Lessons

Lesson #0

Co-maintainership doesn’t work

having core volunteers and expecting them to spread the load (one does fb, other email and website, etc.) didn’t work

when we embraced one pre-assigned guy per months things started to go way better (less late announcements, etc.)

Lesson #1

We accidentally offended several communities

We’re sorry

can’t really give more details because it might offend others but just keep this in mind when doing social events

Lesson #2

It takes time

as you can see making those funny pictures can take a while

Lesson #3

Having checkpoints along a complex challenge is a good thing

We stop to provide hints to make sure no one is stalled. People tend to regroup at these stages and still make progress.

Tools and techniques

There are no single tools or techniques

The END
joking aside, people tend to use various tools, he are what I decided to highlight (I don’t speak on behalf of others)

Learn a scripting language

Accumulate snippets of codes and re-use them, expand them over time

wait for it

but preferably python3

Use ipython and ipython notebook

if you’ve never used notebook, you GOT to give it a try

No plain gdb

fancy .gdbinit or p3da

From time to time try radare2 instead of a pirated IDA Pro

Its getting there. I presented a full crackme type challenge using it.

Wireshark is da bomb and learn tshark

Especially -T fields -e data and then some piping into python with fileinput module for further processing.

Web: Burp or Owasp ZAP

Its been a while since I’ve done web so these might not be the best

Use vagrant for disposable VMs

Learn to use a powerful text-editor like vim, sublime or emacs

Preferably vim

Learn to use the developers tools of your browser

Most of the modern web challenges can be done that way. The 'copy as curl' chrome command is awesome
